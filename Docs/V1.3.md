# Schematic V1.3 Documentation

## 1. Overview
The V1.3 board keeps the dual-processor architecture (MSPM0G3507SPTR and ESP32-S3 WROOM 1) while removing the battery charging and battery switching/load-sharing circuitry introduced previously. This revision returns the design to a wired-power-only approach and simplifies the power path for the current build phase.

## 2. Core Components
The main processing and connectivity elements remain unchanged:
- **Primary Microcontroller (MCU):** Texas Instruments MSPM0G3507SPTR.
- **Secondary Microcontroller:** **ESP32-S3 WROOM 1** module.
- **Inter-Processor Bridge:** Communication bridge between MSPM0 and ESP32 is established.
- **USB-to-UART Controller:** CP2102N for serial communication and ESP32 auto-boot support.
- **Accelerometer:** LIS2DH for motion sensing.

## 3. Power Subsystem (Revision Change)
The V1.3 power subsystem removes all battery-management-specific blocks:
- **Removed:** Li-ion charging circuit.
- **Removed:** Battery switching/load-sharing circuit.
- **System Power:** Board is powered from USB input and regulated to 3.3V through the **LM1117** linear regulator.

## 4. Interfaces and Connectors
- **USB Interface:** USB-C receptacle for power and data.
- **ESP32 Auto-Boot:** DTR/RTS auto-boot circuitry retained.
- **Battery Interface:** No battery charging or battery switching path in this revision.

## 5. Status, Flashing, and Errata
- **Schematic Status:** Finalized for PCB plotting.
- **Flashing:**
	- ESP32-S3 programs via USB-C using the CP2102N auto-boot circuit.
	- MSPM0 requires an **ST-Link** (or compatible SWD programmer).
- **Errata:** No known issues documented for V1.3 at this time.
