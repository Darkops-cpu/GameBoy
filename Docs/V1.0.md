# Schematic V1.0 Documentation

## 1. Overview
The V1.0 board serves as the main processor module for the overarching design. It handles the core logic, processing, and acts as the central hub before integration with the rest of the hardware components.

## 2. Core Components
The board integrates several key components to achieve its processing and connectivity goals:
- **Main Microcontroller (MCU):** Texas Instruments `MSPM0G3507SPTR` in a 48-LQFP package.
- **USB-to-UART Controller:** `CH340E` for serial communication and debugging.
- **Accelerometer:** `LIS2DH` for motion sensing and orientation detection.
- **Clock:** 8MHz external crystal oscillator for precise timing.

## 3. Power Subsystem
The power architecture is designed to step down a standard 5V USB input to the 3.3V logic level required by the MCU and peripherals.
- **Power Input:** Micro USB (5V source).
- **ESD Protection:** `USBLC6-2SC6` on the USB D+/D- lines to protect against electrostatic discharge.
- **Voltage Regulation (LDO):** `SPX3819M5-L-3-3` Low Dropout Regulator to provide a stable 3.3V supply to the system.

## 4. Interfaces and Connectors
As of V1.0, the primary interface is the Micro USB port for power and UART communication. Other external user interfaces (such as screens or buttons) are currently omitted on this revision and will be included once the board is fully integrated into the larger chassis/design.

## 5. Known Issues, Flashing, and Errata
> **Warning: Programming Interface**
> The only significant caveat in this revision involves the programming process. If not done correctly, the board will fail to program.
> - An **ST-Link** (or compatible SWD programmer) is required to flash the `MSPM0G3507SPTR` microcontroller.
> - Ensure proper SWD (Serial Wire Debug) connections (SWDIO, SWCLK, GND, 3V3) are made from the ST-Link to the corresponding pins/pads on the board.
