# Schematic V1.2 Documentation

## 1. Overview
The V1.2 board builds upon the dual-processor architecture of V1.1 (MSPM0G3507SPTR and ESP32-S3 WROOM 1) by introducing full battery power management. This revision features a dedicated Li-ion battery charging and discharging circuit, alongside an automatic power switching mechanism to allow the board to function seamlessly when plugged into USB or running directly off the battery.

## 2. Core Components
The primary processing and connectivity components are carried over from V1.1:
- **Primary Microcontroller (MCU):** Texas Instruments MSPM0G3507SPTR.
- **Secondary Microcontroller:** **ESP32-S3 WROOM 1** module.
- **USB-to-UART Controller:** CP2102N for serial communication and debugging with auto-boot support for the ESP32.
- **Accelerometer:** LIS2DH for motion sensing.

## 3. Power Subsystem (Major Update)
The V1.2 revision completely revamps the power subsystem to support untethered operation:
- **Battery Specifications:** Designed for a standard Li-ion battery.
- **Charging Circuit:** Utilizes a standard TP4056-based charging circuit to safely charge the Li-ion battery when USB-C is connected.
- **Power Switching/Load Sharing:** An automatic power-path switching circuit has been implemented using a combination of two BJTs (NPN and PNP) and one P-Channel MOSFET. This ensures that when the USB-C is connected, the system is powered directly from USB while the battery charges, and when disconnected, it transitions smoothly to battery power without interrupting the processors.
- **System Power:** The board continues to use the **LM1117** linear regulator (carried from V1.1) to provide a stable 3.3V supply to the MCUs from either the battery or USB source.

## 4. Interfaces and Connectors
- **Battery Connector:** A terminal block has been added to securely connect the Li-ion battery to the power management circuit.
- **USB Interface:** USB-C receptacle for data and power delivery/charging.
- **ESP32 Auto-Boot:** Retains the DTR/RTS auto-boot circuitry for the ESP32-S3.

## 5. Known Issues, Flashing, and Errata
- **Flashing:** 
  - ESP32-S3 programs via the USB-C port utilizing the CP2102N auto-boot circuit.
  - MSPM0 still requires an **ST-Link** (or compatible SWD programmer) for flashing.
- **Errata:** There are currently no known issues or errata documented for the V1.2 release.
