# Schematic V2.0 Documentation

![Static Badge](https://img.shields.io/badge/FILE_TYPE-DOCUMENTATION-white)


## 1. Overview
The V2.0 board builds upon the initial design by introducing a dual-processor architecture. It retains the core processing capabilities while adding an ESP32-S3 WROOM 1 module to handle advanced connectivity and secondary processing tasks.

## 2. Core Components
Significant updates have been made to the component selection in this revision:
- **Primary/Secondary Microcontrollers:** 
  - The board now features an **ESP32-S3 WROOM 1** module as a secondary processor for enhanced capabilities and wireless communication.
  - (Assuming retention of the original MSPM0G3507SPTR main MCU and LIS2DH accelerometer from V1.0, unless otherwise replaced).
- **USB-to-UART Controller:** Upgraded to the **CP2102N**, providing robust serial communication and debugging, specifically supporting the new ESP32 integration.
- **Clocking:** Relevant crystal oscillators for both MCUs.

## 3. Power Subsystem
The power architecture has been modernized for better compatibility and current delivery:
- **Power Input:** Upgraded to **USB-C** for both power delivery and data connection.
- **Voltage Regulation (LDO):** The primary 3.3V power supply is now driven by an **LM1117** linear regulator to meet the increased current demands of the ESP32-S3.

## 4. Interfaces, Connectors, and Boot Logic
While external displays or heavy peripherals are still omitted in this revision, crucial interface updates have been made:
- **ESP32 Auto-Boot:** The integration of an auto-boot circuit for the ESP32-S3 has been enabled, allowing for seamless programming and resetting without manual button presses via the CP2102N UART lines (DTR/RTS).
- **USB Interface:** Modern USB-C receptacle replaces the older Micro USB.

## 5. Known Issues, Flashing, and Errata
- **Flashing the ESP32-S3:** The auto-boot circuitry handles putting the ESP32-S3 into the correct boot mode automatically during programming.
- **Flashing the MSPM0:** The original requirement of using an **ST-Link** (or compatible SWD programmer) via the SWD interface remains unchanged from V1.0.
- **Errata:** There are currently no known bugs or errata documented for the V2.0 release.
